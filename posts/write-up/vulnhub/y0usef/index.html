Y0usef - easy

LEVEL : Easy
TIME SPEND : 1h30
LINK : y0usef 1

NMAP

Welcome to a new writeup vulnhub.

Let’s start with an nmap to list the services open on the machine.

sudo nmap -A -p- -Pn -T 5 yousef.vuln

As we can see, there are two main services :

PORT Status Banner
22 OPEN OpenSSH 6.6.1p1
80 OPEN Apache http 2.4.10

Let’s check the HTTP service.

Web Application

On the index page there is nothing interesting.
The web application seems to be under construction.

There is also nothing in the source code.
I think that this is a good excuse to FUZZ this application !

FUZZING

Yeah … I will explain this part quite quickly because I lost a lot of time here.

I used a lot of wordlists on this application but none of them seemed to work, I continued with a BURP analysis to see if I missed anything in the POST but nothing either. Then I cried a little and had a lot of coffee.

Then I installed the wordlist of danielmiessler (seclists) because it was looking promising and I was right, after 3 wordlists I finally came across something interesting!

I hope the author did the typping errors on purpose …

Wow excellent now I can finally start!

Nothing is accessible …

After some research I saw that the author made some configuration errors, so I have to add an HTTP header with BURP to be able to access the site!

BRUP

So, let’s add this HTTP header : X-Forwarded-for: localhost

And, it’s working , I can see the login form.

Adminstration

Okay, I don’t have any creds, let’s try somethig trivial :

  • admin
  • admin

Well, finally something simple !

Upload

There is an upload form, as we can see when we upload a file, like a PNG, it showing us the path where the file was uploaded.

That’s an interesting scenario here, after a few test I figured out that we can easly upload a PHP file and change the MIME type using BURP.

just replace application/x-php by image/png.

And if we go to the path of the uploaded file, we can see the PHP is executed !

Now, we just need to upload a reverse shell and we have our access to the machine with the service account.

System

The www-data account is actually the user of the machine (the user flag is accessible), so now we have to find a way to pass root with a privesc.

Enumeration

As the FUZZING I’m not going to show all my enumeration, just know that I used LinEnum, LSE and LinPeas and at the end none of them were useful to me.

I also use this command to found some interesting binary that I might have missed out:

find / -perm -u=s -type f 2>/dev/null

Linux Kernel

Well, after a few minutes I just use uname and searchsploit and I found a kernel privesc !

Download the exploit, compile and run !

Thanks

Thank you y0usef a few mistake but a funny box, It is not often to sees privesc kernel it is always interesting.
